Que es una Variable
El concepto base de la programacion
Una variable es un espacio en la memoria del computador con un nombre, que guarda un valor que PUEDE CAMBIAR mientras el algoritmo se ejecuta.
Piensa en una variable como una caja etiquetada: puedes meter algo, sacarlo y cambiarlo. En PSeInt, toda variable debe declararse antes de usarse, indicando su nombre y tipo de dato.

Declarar variable en PSeInt
Sintaxis: Definir <nombre> Como <tipo>
En PSeInt se usa la palabra Definir seguida del nombre de la variable, la palabra Como y el tipo de dato. Luego se asigna el valor con la flecha <-
La declaracion le dice al algoritmo: "reserva un espacio en memoria con este nombre y este tipo". La asignacion pone un valor en ese espacio. Son dos pasos distintos.

Constante en PSeInt
Un valor que no debe cambiar nunca
PSeInt no tiene una palabra especial para constantes. La convencion es: declararla como variable, asignarla UNA SOLA VEZ y escribir el nombre en MAYÚSCULAS para indicar que es constante.
Aunque tecnicamente PSeInt permite cambiar cualquier variable, el programador decide no tocarla mas. Usar MAYÚSCULAS es una convencion que avisa a quien lee el codigo: "esto no debe cambiar".
PI <- 3.1416 MAX_VIDAS <- 3 IVA <- 0.19
// Constantes: MAYÚSCULAS por convencion

Tipos de dato en PSeInt
PSeInt maneja cuatro tipos: Caracter (texto), Entero (numero sin decimales), Real (numero con decimales) y Lógico (Verdadero o Falso).
El tipo define que clase de informacion guarda la variable y que operaciones puedes hacer con ella. No puedes sumar un Caracter, ni guardar un texto en un Entero.
"Hola" → Caracter25 → Entero9.99 → RealVerdadero → Logico

Algoritmo completo en PSeInt
Todo algoritmo en PSeInt comienza con la palabra Algoritmo seguida del nombre, y termina con FinAlgoritmo. Las declaraciones van al inicio, antes de las instrucciones.
Primero se declaran todas las variables y constantes con Definir. Luego se escriben las instrucciones del algoritmo. Esta guia usa Algoritmo y FinAlgoritmo para mantener la lectura clara desde el diagrama hasta el código.
